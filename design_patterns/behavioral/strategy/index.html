Strategy Pattern

Define a family of algorithms, encapsulate each one, and make them interchangeable

What is the Strategy Pattern?

The Strategy pattern is a behavioral design pattern that enables selecting an algorithm at runtime. Instead of implementing a single algorithm directly, code receives run-time instructions as to which algorithm to use from among a family of algorithms.

The Strategy pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. It lets the algorithm vary independently from the clients that use it.

When to use the Strategy Pattern:

  • When you want to define a class that will have one behavior that is similar to other behaviors in a list
  • When you need different variants of an algorithm
  • When an algorithm uses data that clients shouldn't know about
  • When a class defines many behaviors, and these appear as multiple conditional statements in its operations

Implementation

Here's a JavaScript implementation of the Strategy pattern using a text formatting example:

JavaScript Implementation
// Strategy interface (implicit in JavaScript)
class TextFormattingStrategy {
    format(text) {
        throw new Error("format() method must be implemented");
    }
}

// Concrete strategies
class PlainTextStrategy extends TextFormattingStrategy {
    format(text) {
        return text;
    }
    
    getDescription() {
        return "No formatting applied. Returns the text as-is.";
    }
}

class UpperCaseStrategy extends TextFormattingStrategy {
    format(text) {
        return text.toUpperCase();
    }
    
    getDescription() {
        return "Converts all characters to uppercase.";
    }
}

class LowerCaseStrategy extends TextFormattingStrategy {
    format(text) {
        return text.toLowerCase();
    }
    
    getDescription() {
        return "Converts all characters to lowercase.";
    }
}

class CamelCaseStrategy extends TextFormattingStrategy {
    format(text) {
        return text
            .toLowerCase()
            .replace(/[\s-_]+(\w)/g, (_, p) => p.toUpperCase());
    }
    
    getDescription() {
        return "Converts text to camelCase (e.g., 'hello world' → 'helloWorld').";
    }
}

class TitleCaseStrategy extends TextFormattingStrategy {
    format(text) {
        return text.replace(
            /\w\S*/g,
            (word) => word.charAt(0).toUpperCase() + word.substr(1).toLowerCase()
        );
    }
    
    getDescription() {
        return "Capitalizes the first letter of each word.";
    }
}

class SlugStrategy extends TextFormattingStrategy {
    format(text) {
        return text
            .toLowerCase()
            .replace(/\s+/g, '-')
            .replace(/[^\w-]+/g, '');
    }
    
    getDescription() {
        return "Creates a URL-friendly slug (e.g., 'Hello World' → 'hello-world').";
    }
}

// Context
class TextFormatter {
    constructor(strategy = new PlainTextStrategy()) {
        this.strategy = strategy;
    }
    
    setStrategy(strategy) {
        this.strategy = strategy;
    }
    
    format(text) {
        return this.strategy.format(text);
    }
    
    getStrategyDescription() {
        return this.strategy.getDescription();
    }
}

Using the strategies is simple:

Using the Strategy Pattern
// Client code
const formatter = new TextFormatter();

// Use the default strategy (PlainTextStrategy)
console.log(formatter.format("Hello World")); // Output: Hello World

// Switch to uppercase strategy
formatter.setStrategy(new UpperCaseStrategy());
console.log(formatter.format("Hello World")); // Output: HELLO WORLD

// Switch to camelCase strategy
formatter.setStrategy(new CamelCaseStrategy());
console.log(formatter.format("Hello World")); // Output: helloWorld

Key Components

Strategy (Interface)

Declares an interface common to all supported algorithms. Context uses this interface to call the algorithm defined by the ConcreteStrategy.

ConcreteStrategy

Implements the algorithm using the Strategy interface. In our example, these are the different text formatting classes.

Context

Is configured with a ConcreteStrategy object, maintains a reference to a Strategy object, and may define an interface that lets Strategy access its data.

Benefits

  • Eliminates conditional statements: Reduces complex conditionals by encapsulating related algorithms into separate strategy classes.
  • Provides alternative implementations: Same problem can be handled in different ways based on the context and requirements.
  • Isolates the algorithm from the client: Client doesn't need to know how the strategies are implemented.
  • Makes switching between behaviors easy: You can switch algorithms at runtime.
  • Follows Open/Closed Principle: You can add new strategies without modifying the context.

Examples in Real-World Applications

  • Sorting Algorithms: Different sorting strategies (QuickSort, MergeSort, BubbleSort) can be used depending on the data size and requirements.
  • Payment Processing: Different payment methods (Credit Card, PayPal, Cryptocurrency) can be implemented as strategies.
  • Navigation Systems: Different routing algorithms (fastest, shortest, avoid highways) can be selected at runtime.
  • Data Compression: Different compression algorithms can be used based on speed vs. compression ratio needs.
  • Authentication: Various authentication methods (password, fingerprint, face recognition) can be implemented as strategies.
